
**System Design**
- Designing for **scalability & fault tolerance**: load balancing, horizontal scaling, message queues, caching layers
- **Zero-Trust Architecture** — least privilege, micro-segmentation, continuous verification
- Event-driven & **distributed system design** for log ingestion pipelines (multi-source SIEM at scale)
- API design, rate limiting, and secure service-to-service auth (mTLS, OAuth2, JWT)
- Threat modeling as a first-class step in the design process (STRIDE, attack surface mapping)

**OS Internals**
- **Linux internals**: process scheduling, virtual memory, syscalls, namespaces & cgroups, `/proc` forensics
- **Windows internals**: process injection techniques, PE file structure, registry & service internals — studied both offensively (red team) and defensively (blue team)
- Kernel-level attack surfaces: privilege escalation vectors, rootkits, and kernel exploit mitigation (ASLR, DEP, SMEP/SMAP)
- Reverse engineering binaries down to assembly with Ghidra, correlating disassembly back to OS-level behavior
